### Breakdown

| Severity | Count | Primary Cause |
|---|---|---|
| Errors | 513 | Unused imports (`no-unused-vars`, `unused-imports/no-unused-imports`) |
| Warnings | 357 | Various (missing deps in `useEffect`, prop-types, etc.) |
| **Total** | **870** | |

### Example Pattern

```js
// Before (triggers ESLint error):
import { useState, useEffect, useCallback, useMemo } from 'react';
// useCallback and useMemo never referenced in file

// After (clean):
import { useState, useEffect } from 'react';
```

### Recommended Fix

Run the auto-fixer for unused imports:

```bash
npm run lint:fix
```

For any remaining errors, manually audit each file. The `eslint-plugin-unused-imports` package is already configured in `eslint.config.js`.

> ⚠️ **Bundle Impact:** Unused imports do not meaningfully increase bundle size (tree-shaking handles most cases), but they do increase cognitive load and make the linter noisy, masking real errors.
